# FoodLab AI Licence Decision

No licence file is included in the V1.0 repository. A public GitHub repository does not automatically grant permission to copy, modify, redistribute, or commercially use its contents.

## Option 1: No licence yet

- Source code remains protected by copyright by default.
- Other people may inspect the public portfolio but do not receive an express reuse licence.
- This is suitable while FoodLab AI remains a long-term personal project and future commercial direction is undecided.

## Option 2: MIT Licence

- Permits broad use, modification, distribution, and commercial use.
- Requires preservation of the copyright and licence notice.
- Provides the software without warranty.

## Option 3: Apache License 2.0

- Permits broad use, modification, distribution, and commercial use.
- Includes explicit patent-related terms and contribution conditions.
- Requires preservation of required notices.

## Current recommendation

**No licence yet.**

This keeps V1.0 focused on employment portfolio presentation while preserving the option to choose MIT, Apache-2.0, or another deliberate licence later. A formal licence should be added only after the intended reuse, contribution, patent, and possible commercialization model is decided.
