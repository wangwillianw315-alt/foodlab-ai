# FoodLab AI Portfolio Guide

FoodLab AI is an educational software portfolio demonstrating how food-science domain knowledge can be translated into traceable, testable browser applications.

## Portfolio narrative

The project began as four independently developed specialist tools:

- formulation and product development;
- sensory evaluation;
- shelf-life study planning;
- production quality monitoring.

V1.0 adds a unified Portal, shared lifecycle identifiers, and explicit JSON transfers so the modules can demonstrate one end-to-end product workflow without prematurely merging their internal stores or introducing a database.

## Suggested interview demonstration

1. Start at the Portal and explain the product lifecycle.
2. Show a Product Development brief and formula version.
3. transfer a shortlisted formula to Sensory.
4. review aggregate sensory evidence and transfer the selected sample to Shelf Life.
5. review a storage study and transfer confirmed planning limits to QA.
6. show linked standards, batch monitoring, transfer history, tests, and scientific disclaimers.

## Engineering evidence

- Five independently runnable React/TypeScript applications.
- Versioned JSON contracts and shared identifiers.
- 242 application tests across 34 files.
- Contract validation for three transfers and four schemas.
- Five passing production builds.
- Local-first privacy boundaries and metadata-only transfer history.

The project uses synthetic or illustrative data. It is not described as commercially validated, regulator-approved, laboratory-certified, or used by real customers. Scientific outputs remain planning or demonstration evidence that requires appropriate validation and professional interpretation.
